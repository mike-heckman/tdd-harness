# Phase 06: Violet (Review & Signoff)

## Description
A final validation phase triggered after the implementation and coverage guardrails pass. The Harness invokes a stateless Review Sub-Agent acting as a Senior Code Reviewer to evaluate the code against the original Task Definition of Done (DoD). If approved, the task is marked complete. If rejected, the Harness transitions the state back to the Green Phase so the primary agent can address the missing requirements.

## Permissions
- `src/`: Read-Only (`ro`)
- `test/`: Read-Only (`ro`)

## Available Tools
- `search_symbols` (MCP)
- `get_symbol_source` (MCP)
- `get_file_content` (MCP)

## Context Payload
*(Note: This payload is fed to the stateless Review Sub-Agent, not the primary execution agent)*
1. Original Task `Context` & `Success Criteria` (Definition of Done)
2. A unified `diff` of all files modified during the task execution (generated by comparing the current working files against their `.tdd-harness/backups/<instance_id>/` copies, ensuring crash-safe isolation).
3. A summary list of touched files
*(The Sub-Agent uses its tools to investigate the codebase if the diff lacks sufficient context to make a ruling).*
